Absolute, Relative and Fixed Positioning

April 10, 2014

For my first technical blog post, I'm choosing to write about positioning. I've chosen this topic because it gave me the most trouble while attempting to complete the Week 1 challenges. I thought this would give me a good reason to research and write-up my findings, which will hopefully not only help me but also help all of you!


To start off, each element has a position value of "static." These elements are always positioned according to the normal flow of the webpage, regardless of properties that may try to change the positioning (left, right, etc.). However, static elements can still run into other elements that are positioned differently. My interpretation is that if every element has a style of "position: static" then nothing on the page will overlap. Of course, the page probably won't be too visually appealing.


Absolute elements are positioned based their first parent element that has a position other than "static." If no element has a different position, the <html> tag will be considered to be the first parent element. I consider that to mean that the element will then be positioned relative to the beginning of the page. Absolute positioning was the most confusing thing for me to work around, because I kept having trouble remembering what the "absolute" position of the element was actually relative to (that's confusing just to type). My opinion is that the way to deal with this is to only use absolute positioning for sections or divs where every element will be positioned as something other than static, but my opinion may change as I work more with positioning.


Relative positioning will position an element relative to where it would be if its position was "static." Relative positioning seems to be the most intuitive positioning to me. If you assign "position: relative" to an element and then give it a property of "left: 20px" then the element will have 20px of space to the left of where it would be if it was a static element. If every other element on the page is static, relative positioning is very easy to implement.


Fixed positioning is extremely useful. Assigning an element to have "fixed" position means that it will not move from that location on the user's browser, no matter how much scrolling the user does. This would be useful for a navigation bar at the top of a page, which many sites have; the bar always stays at the top so the user can easily navigate within the website. For example, Facebook has its navigation bar, which contains a search box and link to the user's profile, among other things, set with a fixed position.
